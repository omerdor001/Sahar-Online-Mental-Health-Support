### 🧠 Sahar: Online Mental Health Support

**Project Description**

Sahar is an online mental health support system designed to assist volunteers and system administrators in managing conversations with help-seekers. The system uses AI algorithms to provide real-time risk analysis, conversation summaries, and a detailed explanation of risk factors. This allows volunteers to respond effectively, and in high-risk situations, the system can send immediate mobile alerts to emergency responders.

**📌 Use Cases**

The system supports several key use cases for managing and analyzing conversations:

* **🔍 Conversation Filtering**: Volunteers and system administrators can filter conversations using a unique identifier to quickly locate and review specific conversations.
* **📊 Real-Time Chat Risk Analysis**: The system continuously monitors and analyzes ongoing chat conversations to evaluate the risk level (low, medium, or high) in real-time. This assessed risk is displayed to the volunteer and triggers alerts when necessary.
* **📝 Conversation Summary**: Volunteers can view a summary of any conversation from the call history or during an active session. The summary is generated by a dedicated AI algorithm.
* **⚙️ Algorithm Configuration Management**: Volunteers can configure and manage which AI algorithm functionalities are active, such as risk level detection and explanation generation.
* **🌐 Fetch Data from LivePerson API**: The system fetches conversation data from the LivePerson API every 5 seconds to monitor and detect high-risk conversations.
* **📂 Access Historical Analysis Data**: System administrators can manually connect to the server to pull historical analysis data, including log files or a database file, from a specific timeframe for offline evaluation of the algorithm's performance.

**🏛️ System Architecture**

The system consists of a `ServerProject` (backend) and a `ClientProject` (frontend) hosted on a BGU server.

* **ServerProject**: This backend project handles server-side logic, AI processing, and API communication. It is built with a **Flask Python** application, a **Gunicorn** server, and an **Nginx** web server. It uses a single **SQLite database** to save relevant persistence information, primarily for tracking conversation predictions.
* **ClientProject**: The frontend is a **React.js** application that uses the **Material UI** library. Nginx serves these static web pages via HTTPS.
* **🧩 LLM Models**: Large language models are deployed on the server for conversation analysis. Different models are used for specific tasks, including a **GSR/IMSR model** for general and immediate suicidal risk, a **summarization model** to summarize conversations, and an **explanation model** to explain why a high-risk prediction was made.
* **📜 Logs**: The system uses two types of logs: system logs to report on actions taken within the system and cache logs for debugging purposes.
